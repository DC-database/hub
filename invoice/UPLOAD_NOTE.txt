IBA System Patch 11.1.6 — Summary Note Vendor Note Index Backfill

Upload/replace only these files:

1) app.js
2) index.html
3) version.json
4) js/app-data-cache.js

Scope:
- Invoice Management only.
- WorkDesk is not changed.
- Inventory is not changed.

What changed:
- Summary Note no longer starts by downloading full invoice_entries.
- Summary Note reads the small invoice_note_index reference list first.
- It exact-reads only the invoice records referenced by the note index.
- When a current note is indexed, previous-note lookup can backfill only the same vendor's PO folders, not all invoice_entries.
- If the note is old and no index exists yet, the system asks before doing a one-time legacy full scan. That one-time scan builds the note index so future Summary Note searches are light.
- Invoice Entry / Batch Entry / Summary Note saves continue updating invoice_note_index.

Expected test:
1) Hard refresh Chrome.
2) Open Summary Note.
3) Opening the page should not show the full invoice_entries warning.
4) Generate Summary using a newly indexed note should use invoice_note_index + exact reads.
5) For old notes not yet indexed, the system may ask for one-time migration before using the old data.

Firebase rules:
- No new rule is required if invoice_note_index was already added:
  "invoice_note_index": {
    ".read": true,
    ".write": true,
    ".indexOn": ["lastUsedAt", "count", "text"]
  }
